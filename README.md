# VoxHire — AI Interview Practice

VoxHire AI is a full-stack web application designed to simulate high-pressure job interviews using low-latency, conversational AI. Unlike traditional text-based chatbots, VoxHire utilizes real-time Speech-to-Speech (S2S) technology to allow candidates to practice speaking naturally with an AI interviewer.

It combines:
- **Firebase Auth + Firestore** for user sessions & saving interviews
- **Google Gemini / OpenAI (via `ai` package)** to generate interview questions
- **VAPI** for voice-enabled real-time interview sessions

---

## 🚀 Key Features

- ✅ **Generate interview questions** based on role, experience level, tech stack, and focus (behavioral vs. technical)
- ✅ **Voice-powered mock interview** (real-time transcription & speech)
- ✅ **Autogenerated feedback** based on your interview transcript
- ✅ **User sign-in / sign-up flow** using Firebase Authentication
- ✅ **Interview history stored in Firestore** for review and retakes

## 🛠️ Local Development

### 1) Install dependencies

```bash
npm install
```

### 2) Configure environment variables

Create a `.env.local` file at the project root and set the following values:

```env
# Firebase (client)
NEXT_PUBLIC_FIREBASE_API_KEY=
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=
NEXT_PUBLIC_FIREBASE_PROJECT_ID=
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=
NEXT_PUBLIC_FIREBASE_APP_ID=
NEXT_PUBLIC_FIREBASE_MEASUREMENT_ID=

# Firebase (admin - server)
FIREBASE_PROJECT_ID=
FIREBASE_CLIENT_EMAIL=
FIREBASE_PRIVATE_KEY=

# VAPI (web client + workflow)
NEXT_PUBLIC_VAPI_WEB_TOKEN=
NEXT_PUBLIC_VAPI_WORKFLOW_ID=
```


### 3) Run the dev server

```bash
npm run dev
```

Open http://localhost:3000 in your browser.

## 🧭 App Structure

- `app/` — Next.js App Router pages + API routes
- `components/` — UI components (Agent, AuthForm, InterviewCard, etc.)
- `firebase/` — Firebase client + admin initialization
- `lib/` — SDK wrappers, actions, and utilities
- `types/` — TypeScript types for the app

## 🧩 Notes / Customization Ideas

- Swap the `generateText` model in `app/api/vapi/generate/route.ts` to try different LLMs (Gemini, GPT-4o, etc.)
- Tune your VAPI voice workflow by editing the workflow definition in the VAPI dashboard
- Add additional question categories or persistence (e.g., analytics, user settings, saved interviews)
